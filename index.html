Comparing images with different subdiv methods

I have tested several methods on a same fbx model

  1. Current method, baked subdiv from blender
  2. Vrmesh converted from vrscene generated from method 1
  3. Alembic file exported from blender with osd properties
  4. Vrscene generated from blender w/o baking the geometry (fbx-spliter) with osd properties
  5. Vrscene generated from blender w/o baking the geometry (fbx-spliter) rendered with GeomStaticSmoothedMesh node
  6. Vrscene generated from max with osd properties
  7. Rendered inside 3ds max with openSubDiv modifier applied

Notes:

  • Number 7 is for reference only
  • Number 5 is showing worst results in the Box test, it seems is not using the OSD algorithm for subdivinding the surface, although UV looks fine, and in some of the other examples the difference in geometry is not that huge
  • There is no difference between the current method (1) and the vrmesh method (2) so one of these is redundant
  • Number 4, vrscene generated from fbx-splitter. This is showing UV distortion on some models, while a vrscene converted from max (6) does not have those issues
  • So far (3) and (6) have the most promissing results
